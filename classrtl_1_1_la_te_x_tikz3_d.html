Detailed Description

LaTeX export of high quality vector graphics using the Tikz package - 3D scene rendering into 2D drawing.

This class is used to aggregate graphic primitives to be rendered into a PDF format. The rendering order is determined using traditional visibility testing and do not require any input from the user side. View of the scene can be set using regular translation-rotation-projection scheme.

More complicated intersections of polygonal faces may result in visual artifacts appearing as darker lines. This happens, when two polygons of the same color are next to each other and is a rendering bug in many .pdf viewers caused by an antialiasing algorithm. If perfect results are desired, the vector graphics can be converted to raster with a suitable software (e.g. Gimp), or the generated code can be manually edited to merge the neighbouring polygons of the same style.
